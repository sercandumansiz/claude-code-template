# Documentation Rules

- Write in clear, concise English
- Use tables for structured comparisons (e.g., scope comparisons, feature matrices)
- Include code examples for every configuration option
- Link to official Claude Code documentation (https://code.claude.com/docs) rather than maintaining local copies
- Keep consistent heading hierarchy: h1 for title, h2 for sections, h3 for subsections
- Use fenced code blocks with language identifiers (json, bash, markdown)
- Avoid duplicating official docs — link to the canonical source instead
